Троичные риск-решения
для реальных данных

Когда данных не хватает, бинарщина и вероятности часто заставляют гадать. tri-risk честно говорит M и возвращает need_list, что именно добрать для уверенного решения.

Получить API-ключ Открыть документацию Локально уже работает: /v1/risk, /v1/usage, /v1/openapi.json

Троичная логика 0/M/1

M — не вероятность и не «полуправда». Это честная неизвестность.

Объяснимость

Возвращаем label и need_list: что не хватает, чтобы решить.

Просто интегрировать

Один POST, JSON на вход и выход. Работает рядом с твоими правилами.

Ключи и лимиты

X-API-Key + rate limit. Без OAuth-цирка.

Usage

/v1/usage показывает расход лимитов по ключу.

OpenAPI

/v1/openapi.json для SDK/Swagger/Postman.

Один запрос — полное решение

Пример: часть полей неизвестна → получаем M и список, что нужно добрать.

curl -X POST http://127.0.0.1:7444/v1/risk \
  -H "X-API-Key: pro_key_999" \
  -H "Content-Type: application/json" \
  -d '{
    "env": {
      "income": "1",
      "docs": "M",
      "fraud": "0",
      "history": "M"
    }
  }'
{
  "decision": "M",
  "label": "REVIEW",
  "need": 3,
  "need_list": ["docs","history"]
}

Документация

/v1/docs /v1/openapi.json /v1/usage

На проде эти ссылки будут на твоём домене. Сейчас они указывают на локальный proxy 7444.
